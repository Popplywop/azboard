azboard v0.1.0

A fast, keyboard-driven terminal UI for Azure DevOps. Review pull requests, manage work items, and stay in your terminal.

Installation

curl -fsSL https://popplywop.github.io/azboard/install.sh | sh

Installs to ~/.local/bin and adds it to your PATH automatically. Set INSTALL_DIR to change.

iwr https://popplywop.github.io/azboard/install.ps1 | iex

Installs to %LOCALAPPDATA%\azboard and adds to PATH.

Download from the releases page:

PlatformFile
Linux x86-64azboard_*_linux_amd64.tar.gz
Linux ARM64azboard_*_linux_arm64.tar.gz
macOS Intelazboard_*_darwin_amd64.tar.gz
macOS Apple Siliconazboard_*_darwin_arm64.tar.gz
Windows x86-64azboard_*_windows_amd64.zip

Requires Go 1.21+.

git clone https://github.com/Popplywop/azboard
cd azboard
go build -o azboard .
mkdir -p ~/.local/bin
mv azboard ~/.local/bin/azboard

Quick Start

1. Install azboard using one of the methods above.

2. Create a config file at ~/.config/azboard/config.json:

{
  "org_url": "https://dev.azure.com/your-org",
  "project": "your-project"
}

3. Authenticate with Azure CLI:

az login

4. Launch: azboard

5. Press R to open the repo picker, select repos with Space, press Ctrl+S to save.

Configuration

All settings are read from ~/.config/azboard/config.json.

Full Example

{
  "org_url": "https://dev.azure.com/your-org",
  "project": "your-project",
  "auth_method": "azcli",
  "repos": ["my-api", "my-frontend"],
  "work_item_types": ["User Story", "Bug", "Task"],
  "default_merge_strategy": "squash",
  "area_path": "Project\\Team",
  "auto_refresh_seconds": 30
}

Config Reference

FieldRequiredDefaultDescription
org_urlYes--Azure DevOps URL. Supports dev.azure.com and visualstudio.com.
projectYes--Project name. Can be embedded in org_url.
auth_methodNoazcliazcli or pat.
patIf PAT--Personal access token with Code and Work Items scopes.
reposNo[]Repos to load on startup.
work_item_typesNo5 default typesWork item types to display.
default_merge_strategyNosquashsquash, merge, rebase, or semilinear.
area_pathNo--Filter work items to an area path.
auto_refresh_secondsNo0Background refresh interval in seconds. Minimum 10 if enabled. Set to 0 to disable.

Authentication

Azure CLI (recommended)

Install the Azure CLI and run az login. Tokens are obtained and refreshed automatically.

Personal Access Token

Generate a PAT in Azure DevOps with Code (Read & Write) and Work Items (Read & Write) scopes.

{
  "auth_method": "pat",
  "pat": "your-token-here"
}

Features

⚒ Pull Requests

List, filter, create, merge, abandon, vote, comment. Full PR lifecycle in the terminal.

🔍 Code Review

File tree, unified diffs with line numbers, per-iteration comparison, inline comments.

☰ Work Items

Browse stories, bugs, tasks, epics. Transition states, add comments, link to PRs.

⚙ Repo Selection

Interactive multi-select picker with search. Save to config with Ctrl+S.

⚡ Caching & Auto-Refresh

In-memory caching for instant navigation. Optional background polling keeps data fresh.

Caching & Auto-Refresh

azboard caches API responses in memory to reduce latency on repeated views. Caches are automatically invalidated when you perform mutations (merge, vote, comment, state change) or press r to manually refresh.

To enable automatic background refresh, set auto_refresh_seconds in your config:

{
  "auto_refresh_seconds": 30
}

Data updates silently without disrupting your current view. Set to 0 (or omit) to disable. Minimum value is 10.

Verifying Releases

All artifacts are signed with cosign keyless signing via GitHub Actions OIDC.

cosign verify-blob \
  --certificate         azboard_0.0.1_linux_amd64.tar.gz.pem \
  --signature           azboard_0.0.1_linux_amd64.tar.gz.sig \
  --certificate-identity-regexp "https://github.com/Popplywop/azboard" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  azboard_0.0.1_linux_amd64.tar.gz

The .sig and .pem files are attached to each release. If cosign is installed, install.sh verifies automatically.
